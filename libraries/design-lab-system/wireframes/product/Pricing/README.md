# Pricing Wireframe

This Wireframe tests three different UX hypotheses against the same subscription and token-entitlement states.

- **Compare plans** keeps the plan matrix visible for deliberate buyers.
- **Recommended path** reduces choice density and gives Medium the strongest hierarchy.
- **Guided upgrade** prioritizes current usage and the next available action for existing customers.

Use Dev mode to switch saved states or create a custom condition with radio, Checkbox, and Slider controls. User flow shows the authored transition graph; selecting a node updates the state, while Preview state returns to the rendered screen.

The review question is not which card style looks best. It is which information architecture helps a buyer understand the next valid action with the least ambiguity.
